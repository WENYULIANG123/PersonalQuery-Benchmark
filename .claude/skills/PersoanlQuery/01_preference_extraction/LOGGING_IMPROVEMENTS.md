## 🎯 预期益处

### 问题诊断

现在可以识别:
1. **API 限流**: 如果看到重复的 429 错误
2. **JSON 格式错误**: 看到具体的解析错误位置
3. **网络问题**: ConnectionError, Timeout 等
4. **数据质量**: 评论过短或缺失
5. **并发问题**: 多个产品同时失败

### 数据分析

- 对比成功和失败产品的错误模式
- 识别是否存在特定产品类型的失败
- 追踪错误是否来自数据或系统

## 📌 如何使用这些日志

### 1. **实时监控**
```bash
# 运行脚本并观看日志流
python3 01_extract_preferences_v2_with_aspects.py \
  --input-file reviews.json \
  --output-dir output | tee extraction.log
```

### 2. **离线分析**
```bash
# 过滤特定 ASIN 的日志
grep "\[B001BF1BTC\]" extraction.log

# 查看所有错误
grep "❌" extraction.log

# 查看成功的产品
grep "✅" extraction.log
```

### 3. **统计分析**
```bash
# 统计错误类型
grep "❌" extraction.log | cut -d: -f2 | sort | uniq -c

# 统计成功的产品
grep "✅ 提取完成" extraction.log | wc -l
```

## 🔧 技术细节

### 日志级别

- **INFO** (默认): 正常流程和进度
- **DEBUG**: 详细的处理步骤 (当需要时启用)
- **ERROR**: 异常和失败信息
- **SUCCESS**: 成功操作的确认

### 日志格式

```
[时间戳] [ASIN] [符号] 消息
```

例如:
```
[2026-03-18 12:00:00] [B001BF1BTC] ✅ 提取完成: 4 个方面
```
