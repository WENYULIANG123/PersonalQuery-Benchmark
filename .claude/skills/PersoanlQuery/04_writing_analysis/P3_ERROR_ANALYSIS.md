## 1️⃣ 标点符号错误 (最常见)

### A. 缺少逗号

**模式**: 连接词 (and, but, so, as) 前缺少逗号

#### 例子1 - Oxford Comma
```
原文: "...nearby so I thought to check Amazon, where..."
修正: "...nearby, so I thought to check Amazon, where..."

分析: 复杂句式中，所有从句前都应该有逗号
规则: Oxford style - 列表和并列从句前都应有逗号
```

#### 例子2 - 复杂修饰
```
原文: "...high quality. The dye is evenly applied and is not faded."
修正: "...high quality. The dye is evenly applied and is not faded,"

分析: 多个独立陈述句之间的逻辑连接
```

#### 例子3 - 条件从句
```
原文: "...jewelry pieces and they are protected by plastic..."
修正: "...jewelry pieces, and they are protected by plastic..."

分析: 并列独立句前的连接词需要逗号
规则: FANBOYS规则 (for, and, nor, but, or, yet, so)
```

**总数**: ~8个改动涉及添加逗号

**模式识别**:
- 主要集中在复杂句式
- 多个从句或并列结构
- 用户原文中逗号使用不足

---

### B. 多余空格删除

**模式**: 单词间多个空格 → 单个空格

```
原文: "These clasps (I got the 5 pack)  are a little large"
        (注意两个空格)
修正: "These clasps (I got the 5 pack) are a little large"
        (单个空格)
```

**频率**: 每条评论有3-6个多余空格  
**原因**: 用户在输入时不小心按了多次空格  
**严重程度**: 低 (纯格式问题)

---

### C. 引号处理

**模式**: LLM在评论开头和结尾添加引号

```
原文: I am a huge fan of toggle clasps. They're about...
修正: "I am a huge fan of toggle clasps. They're about..."
      (首尾添加引号)
```

**频率**: 每条评论都有 (开头1个 + 结尾1个)  
**问题**: 这可能不是修正错误，而是LLM的输出格式化  
**需要人工验证**: 是否真的是用户错误

---

## 2️⃣ 语法错误 (较少但重要)

### A. 虚拟语气

**例子**:
```
原文: "If I was to..."
修正: "If I were to..."

规则: 虚拟条件句必须使用were (即使主语是I, he, she)
频率: 罕见 (仅在1-2条评论中出现)
严重程度: 高 (标准英文语法错误)
```

**语法规则**:
```
虚拟语气模式:
  if + were + 主语 + 动词
  vs 错误用法:
  if + was + 主语 + 动词

英文考试中常见的考点
```

---

### B. 从句修饰的搭配

**例子**:
```
原文: "...colors in a color palette that I will get a lot of use"
修正: "...colors in a color palette that I will get a lot of use out of"

分析: 短语动词 (phrasal verb) 的完整性
类型: 习惯用法的补全
```

---

## 3️⃣ 格式/风格问题 (低优先级)

### A. 破折号标准化

```
原文: " - "  (连字符)
修正: "–"   (en-dash)

说明: Unicode标准化
影响: 排版效果
```

### B. 引号风格

**现象**: 
- 普通引号 (") → 智能引号 ("")
- 目的: 排版标准化

**问题**: 用户原文使用的引号风格是否有误？还是仅是LLM的输出格式?

---

## 4️⃣ 大小写调整 (少量)

### 例子:
```
原文: "...then colored with the tattered ribbon"
修正: "...then colored with the Tattered Ribbon"

分析: 可能识别为标题或专有名词
问题: 这是否是用户的原始错误？
```

## 📊 按严重程度分类

### 🔴 高严重度 (真实错误)
- **虚拟语气** (was → were)
- **关键短语动词缺失** (get a lot of use → get a lot of use out of)
- **语法结构错误**

**数量**: 2-3个  
**特点**: 影响语义理解和英文规范性

### 🟡 中等严重度 (可选修正)
- **缺少逗号** (尤其在复杂句式)
- **标点符号标准化**
- **多余空格**

**数量**: 8-10个  
**特点**: 影响可读性和专业性，但不影响理解

### 🟢 低严重度 (风格问题)
- **引号添加** (可能是LLM行为，不是用户错误)
- **大小写标准化**
- **破折号风格**

**数量**: 3-4个  
**特点**: 纯格式/风格改动，可能不是真实错误
