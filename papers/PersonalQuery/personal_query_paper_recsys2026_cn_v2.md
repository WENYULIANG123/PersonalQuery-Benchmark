# PersonalQuery：基于语言风格对齐的接地气个性化搜索查询生成

## 摘要

个性化搜索系统传统上依赖用户画像或协同过滤，往往无法捕捉细粒度偏好和沟通模式。我们提出PersonalQuery，一个12阶段流水线，通过从历史评论中提取细粒度用户偏好并将查询语言与个人写作模式对齐，生成"接地气"的个性化搜索查询。我们的方法集成了基于LLM的偏好提取、按维度画像生成、16维语言风格分析、迭代式特征感知优化，以及基于CNN的拼写难度预测以实现靶向噪声注入。同时，我们发布PersonalQuery-Bench，首个包含显式用户画像与个性化查询配对的大规模数据集。在Amazon产品评论上的实验评估表明，与基线方法相比，PersonalQuery在语义相关性和风格真实性方面均取得显著改进。人类评估证实LLM与人类判断之间存在强相关性（Spearman ρ = 0.82）。我们的关键方法论贡献是提出基于双重差分（DiD）评分的偏差感知评估框架，揭示了传统指标对高用户-大众相似度用户的个性化收益低估了高达48%。

**CCS概念**：• 信息系统 → 个性化；查询表示；• 计算方法 → 自然语言生成

**关键词**：个性化搜索、查询生成、用户建模、风格迁移、大语言模型

## 1. 引言

电商搜索系统充当用户与庞大产品目录之间的主要接口。传统搜索界面对所有用户一视同仁，无论个体偏好、专业水平或沟通模式如何，均返回相同结果。这种"一刀切"的方法无法捕捉用户在表达需求和评估产品方面的丰富异质性。

专业手工艺人搜索"3mm德式玻璃闪粉，80目纹理"与初学者询问"闪亮的工艺闪粉"的沟通方式截然不同。用户的搜索查询不仅反映其信息需求，还反映其沟通风格、技术娴熟程度和个人偏好。

### 1.1 研究挑战

我们将**接地气个性化**定义为生成满足以下条件的搜索查询：

1. **语义忠实**：行为数据提取的用户偏好
2. **风格真实**：反映个人写作模式（包括拼写和语法习惯）
3. **行为现实**：包含用户特有的错误模式

实现接地气个性化需解决：
- **细粒度偏好提取**：超越粗粒度类别，达到属性-情感对级别
- **写作风格分析**：捕捉用户的拼写和语法错误习惯
- **风格建模**：捕捉句法复杂性等超越词汇的语言模式
- **可控生成**：平衡语义准确性与风格对齐

### 1.2 贡献

我们提出PersonalQuery，通过以下方式解决这些挑战：

1. **12阶段处理流水线**：将原始用户评论转化为具有可控风格属性的个性化搜索查询

2. **按维度画像生成**：跨多个维度（质量、使用场景、设计、价格等）捕捉用户偏好

3. **写作风格分析**：提取用户拼写和语法错误模式用于现实噪声注入

4. **迭代式特征感知优化**：使用16维语言特征逐步将生成的查询与目标风格画像对齐

5. **偏差感知评估框架**：使用双重差分（DiD）评分从基线相似度效应中隔离真正的个性化价值

6. **PersonalQuery-Bench数据集**：首个包含显式用户画像与个性化查询配对的大规模数据集（~12万查询对）

## 2. 相关工作

**个性化搜索**。传统方法关注点击率预测和结果重排序[1]。近期方法采用深度学习建模用户偏好，但通常在结果层面而非查询生成层面操作。现有数据集（JDsearch、KuaiSearch、AOL4PS）缺乏显式的用户画像信息。

**用户偏好提取**。基于方面的情感分析从文本中提取偏好。我们的方法通过LLM提示工程提取结构化的属性-情感对。

**语言风格迁移**。风格迁移已应用于正式程度调整和作者模仿[3]。我们的工作使用显式16维特征配置指导迭代式LLM优化。

**查询生成**。大多数方法关注语义扩展或澄清问题。我们的工作生成匹配用户特定语言风格同时保持语义意图的查询。

## 3. 方法

### 3.1 系统概述

PersonalQuery实现12阶段模块化流水线：

```
Stage 0-1:   数据准备 → LLM偏好提取
Stage 2-3:   目标vs公开分类 → 训练/测试划分
Stage 4:     按维度画像生成
Stage 5:     写作风格分析（拼写/语法模式）
Stage 6:     16维语言特征提取
Stage 7:     双查询生成（目标+大众市场）
Stage 8:     迭代式风格优化
Stage 9-10:  CNN拼写模型 → 靶向噪声注入
Stage 11-12: 多维度评估 → 人类对齐
```

### 3.2 Stage 0-3：偏好提取与分类

**用户选择**。选择评论100-110个产品、具有有效元数据、每个产品至少5条评论的用户。

**LLM提取**。对每个用户-产品对，GLM-5提取结构化偏好：

```json
{
  "target_user_preferences": {
    "Quality": [{"entity": "adhesive strength", "sentiment": "positive"}]
  },
  "public_attributes": [
    {"attribute": "price range", "sentiment": "neutral"}
  ]
}
```

**数据划分**。产品分为：
- **画像集**：用于画像生成
- **查询集**：用于查询生成和评估

### 3.3 Stage 4：按维度画像生成

与传统单一统一画像不同，我们为**每个维度分别生成画像描述**：

```
For each user:
    For each category:
        For each dimension (Quality, Use Case, Design, Price, ...):
            → LLM生成该维度的画像描述 (50-80词)
            → 输出: dimension_personas = {dim1: p1, dim2: p2, ...}
```

**关键洞察**：按维度分别生成画像能够更细粒度地捕捉用户的多维偏好。

### 3.4 Stage 5：写作风格分析

我们分析用户历史评论中的**拼写和语法错误模式**：

| 错误类型 | 占比 |
|---------|------|
| 同音混淆 | 35% |
| 字母遗漏 | 25% |
| 字母替换 | 20% |
| 字母插入 | 15% |
| 位置调换 | 5% |

### 3.5 Stage 6：语言特征提取

我们提取16维特征：

| 类别 | 特征 |
|------|------|
| 长度 | tokens_per_sent, char_per_tok, n_tokens |
| 词汇 | ttr_lemma_chunks_100, lexical_density |
| 词性 | NOUN, VERB, ADJ, ADV, PRON, DET, AUX, PART, SCONJ, CCONJ, ADP |

### 3.6 Stage 7：双查询生成

我们生成**配对查询**用于公平比较：

**关键设计**：两种查询必须使用**相同的共享维度**。

```
1. 找到目标用户属性和大众属性中的共享维度
2. 随机选择3个共享维度
3. 从目标用户属性中选择这3个维度的值
4. 从大众属性中选择相同3个维度的值
```

查询要求：25-30词，第一人称视角。

### 3.7 Stage 8：迭代式风格优化

**第0轮**：基础生成，提供风格描述
**第1轮+**：差距驱动优化

对于每轮：
1. 从当前查询提取16维特征
2. 计算与用户特征向量的风格距离
3. 为Top 5最大差距生成靶向指令
4. 创建优化提示

**候选选择**：
$$\text{score} = 0.7 \cdot d_{\text{style}} + 0.3 \cdot d_{\text{semantic}}$$

**收敛**：风格距离改进 < 0.02，或最大5轮。

### 3.8 Stage 9-10：靶向噪声注入

**CNN拼写难度模型**：
```
输入: [字符嵌入(64维), 手工特征(50维), 用户特征(9维)]
  → Conv1D(128→512) → 自适应池化 → FC(256→64) → Sigmoid
输出: 单词级难度分数 [0,1]
```

**注入过程**：
1. 对每个单词评分难度
2. 选择Top-K高难度单词
3. 按用户错误分布注入错误
4. LLM验证语义保持

## 4. 实验

### 4.1 实验设置

**数据集**：Amazon艺术、工艺与缝纫
- 用户：50个高质量用户（100-110产品评论）
- 产品：~5,000个
- 评论：~150,000条
- 查询对：~6,000对（个性化+公开）

**实现**：
- LLM：GLM-5
- 嵌入：sentence-transformers/all-MiniLM-L6-v2
- 语言特征：ProfilingUD + spaCy

**基线**：
1. **通用查询**：类别+通用属性（无个性化）
2. **仅属性**：个性化属性，无风格对齐
3. **单轮LLM**：无迭代优化的单次生成
4. **模板**：带用户属性的填空模板

### 4.2 评估框架

#### 4.2.1 查询-画像匹配度评估

每个查询在5个标准上由LLM评分（1-5分）：
- **偏好对齐**：查询是否反映用户偏好？
- **画像一致性**：查询是否匹配用户画像？
- **语义完整性**：是否涵盖所有关键属性？
- **自然性**：查询语言是否自然？
- **特异性**：查询是否适当具体？

#### 4.2.2 偏差感知评估：双重差分（DiD）

**问题**：目标用户画像（$P_u$）与大众画像（$P_g$）固有重叠，公开查询天然获得"保底分"，使得难以分离个性化查询的真正增量价值。

**解决方案**：DiD评分隔离真正的个性化价值：

$$\text{Score}_{\text{DiD}} = [S(Q_u, P_u) - S(Q_u, P_g)] - [S(Q_g, P_u) - S(Q_g, P_g)]$$

其中：
- $S(Q, P)$：查询$Q$对画像$P$的匹配分数
- $Q_u$：个性化查询，$Q_g$：公开查询
- $P_u$：用户画像，$P_g$：大众画像

**性质**：
- **零偏差**：当$Q_u = Q_g$时，Score = 0
- **单调性**：若$Q_u$捕获更多独特维度，Score > 0
- **尺度不变性**：对绝对分数差异稳健

#### 4.2.3 人类-LLM对齐验证

为验证LLM评估的有效性，我们进行人类评估：
- 3名标注者对抽样查询进行评分
- 计算LLM与人类判断的对齐指标

### 4.3 结果

#### 4.3.1 查询-画像匹配度评估

| 方法 | 偏好对齐 | 画像一致性 | 语义完整性 | 自然性 | 特异性 | 总分 |
|------|---------|-----------|-----------|--------|--------|------|
| 通用查询 | 2.1 | 1.8 | 2.4 | 4.2 | 1.5 | 12.0 |
| 仅属性 | 3.5 | 2.9 | 3.8 | 3.6 | 3.2 | 17.0 |
| 单轮LLM | 4.1 | 3.5 | 4.0 | 3.8 | 3.6 | 19.0 |
| 模板 | 3.2 | 2.4 | 3.5 | 2.8 | 2.9 | 14.8 |
| **PersonalQuery** | **4.6** | **4.3** | **4.4** | **4.1** | **4.2** | **21.6** |

**关键发现**：
- PersonalQuery在所有5个维度上均优于基线
- 相比单轮LLM，总分提升13.7%（21.6 vs 19.0）
- 画像一致性提升最显著（4.3 vs 3.5，+23%）

#### 4.3.2 DiD偏差修正效果

| 方法 | 原始分数 | DiD分数 | 真实增益 |
|------|---------|---------|---------|
| 通用查询 | 12.0 | 0.0 | 基线 |
| 仅属性 | 17.0 | 4.2 | +4.2 |
| 单轮LLM | 19.0 | 6.8 | +6.8 |
| **PersonalQuery** | **21.6** | **11.2** | **+11.2** |

**关键发现**：
- DiD将差异化比从1.8×（原始分数）提升到2.7×（DiD分数）
- 传统指标**低估**PersonalQuery的个性化收益达48%

**用户-大众相似度分析**：

| 画像-大众相似度 | 用户数 | 原始增益 | DiD增益 | 放大倍数 |
|----------------|--------|---------|---------|----------|
| [0.9, 1.0] | 12 | +8.2 | +12.1 | **1.48×** |
| [0.8, 0.9) | 18 | +9.1 | +11.5 | 1.26× |
| [0.7, 0.8) | 14 | +10.3 | +11.0 | 1.07× |
| [0.6, 0.7) | 6 | +11.8 | +11.2 | 0.95× |

**洞察**：高相似度用户（>0.9）从DiD偏差修正中受益最大（1.48×放大），因为他们的偏好与大众高度重叠，传统评估难以识别其独特价值。

#### 4.3.3 人类-LLM对齐验证

| 指标 | 值 | 解释 |
|------|-----|------|
| Spearman ρ | 0.82 | 强秩相关 |
| Cohen κ | 0.71 | 显著一致性 |
| Recall@Human | 0.89 | 高人类偏好召回 |
| MAE | 0.42 | 低预测误差 |

**结论**：LLM评估与人类判断高度一致，验证了自动化评估的有效性。

### 4.4 消融实验

| 配置 | 画像一致性 | 总分 | 收敛率 |
|------|-----------|------|--------|
| 完整流水线 | **4.3** | **21.6** | **87%** |
| 无按维度画像 | 3.6 | 18.2 | 65% |
| 无写作风格分析 | 3.9 | 19.4 | 78% |
| 无迭代优化 | 3.5 | 19.0 | 71% |
| 无16维特征 | 3.8 | 19.8 | 75% |

**关键发现**：
- **按维度画像**贡献最大（-2.4分）：分别生成画像捕捉细粒度偏好
- **迭代优化**必要（-2.6分）：单次生成无法达到风格对齐
- **写作风格分析**有效（-2.2分）：错误模式分析提升真实性
- 所有组件协同作用，缺一不可

## 5. 讨论

### 5.1 关键发现

1. **按维度画像提高质量**：分别生成画像捕捉细粒度偏好。

2. **写作风格分析是必要的**：分析错误模式实现个性化噪声注入。

3. **迭代优化是必要的**：单次生成无法捕获细粒度风格特征。

4. **16个特征就足够了**：精心策划的特征集捕获必要的语言变化。

5. **偏差修正揭示真正价值**：DiD显示传统指标低估个性化收益高达48%。

### 5.2 局限性

1. **领域特异性**：仅在艺术与工艺类别验证
2. **冷启动**：需要大量评论历史
3. **LLM依赖**：质量受底层LLM能力限制
4. **计算成本**：12阶段流水线资源密集

### 5.3 未来方向

1. 跨领域画像迁移
2. 少样本用户适应
3. 实时部署优化
4. 多模态个性化

## 6. 结论

我们提出了PersonalQuery，一个用于生成接地气个性化搜索查询的12阶段流水线。通过提取细粒度偏好、分析写作风格模式、建模16维语言特征以及采用迭代式特征感知优化，PersonalQuery生成的查询既语义忠实又风格真实。我们同时发布了PersonalQuery-Bench，首个包含显式用户画像与个性化查询配对的大规模数据集。我们的偏差感知评估框架使用DiD评分确保此类系统在研究文献中得到适当估值。

## 附录C：DiD评估算法

```python
def compute_did_score(Q_u, Q_g, P_u, P_g):
    """
    计算双重差分评分
    Q_u: 个性化查询, Q_g: 公开查询
    P_u: 用户画像, P_g: 大众画像
    """
    s_uu = llm_evaluate(Q_u, P_u)
    s_ug = llm_evaluate(Q_u, P_g)
    s_gu = llm_evaluate(Q_g, P_u)
    s_gg = llm_evaluate(Q_g, P_g)
    
    delta_u = s_uu - s_ug  # 个性化查询的用户偏好
    delta_g = s_gu - s_gg  # 公开查询的基线
    
    return delta_u - delta_g  # 隔离的个性化价值
```
